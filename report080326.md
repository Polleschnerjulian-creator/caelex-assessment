## EXECUTIVE SUMMARY

Caelex is a full-stack space regulatory compliance SaaS platform built on Next.js 15, PostgreSQL (Neon), Prisma ORM, and NextAuth v5. The platform serves satellite operators, launch providers, and space service companies with compliance assessment, monitoring, and reporting across EU Space Act, NIS2 Directive, and 10 national space law jurisdictions.

## 25. ASTRA AI — DETAILED ARCHITECTURE

### What It Is

ASTRA (AI Space & Technology Regulatory Assistant) is a conversational AI compliance copilot powered by Anthropic's Claude (`claude-sonnet-4-6`). Operates in 3 context modes: `general`, `article` (scoped to a specific EU Space Act article), and `category`/`module`.

### Architecture

**Frontend:**

- `AstraProvider.tsx` — React context (13KB): message list, context mode, conversation ID, `isTyping`, error state, `remainingQueries`. Uses `AbortController` for request cancellation.
- `AstraFullPage.tsx` — Full-page layout (13KB): left conversation list (280px, toggleable), center chat, right tool browser (320px, toggleable)
- EU AI Act Art. 50 consent gate on first visit (German language). Stored in `localStorage` as `"caelex-astra-consent": "true"`.

**Backend (`src/lib/astra/`):**

- `engine.ts` — `AstraEngine` class. Manages Anthropic tool-use loop (max 10 iterations). `max_tokens: 4096`, `temperature: 0.7`.
- `system-prompt.ts` — Builds system prompt with user organization context
- `context-builder.ts` — `buildCompleteContext()` fetches compliance scores, assessment data, active workflows from DB
- `tool-executor.ts` — Executes tool calls (2,046 LOC): article lookup, compliance status, requirement search
- `tool-definitions.ts` — `ALL_TOOLS` array passed to Anthropic API
- `conversation-manager.ts` — Prisma-backed persistence with auto-summarization
- `regulatory-knowledge/` — Static knowledge files: `eu-space-act.ts`, `nis2.ts`, `jurisdictions.ts`, `glossary.ts`, `cross-regulation-map.ts`

**API:** `POST /api/astra/chat` — Auth + org required, rate-limited (`assessment` tier, per-user key `astra:${userId}`). All chat events logged to audit trail.

**Fallback:** Without `ANTHROPIC_API_KEY`, `generateFallbackResponse()` returns topic-detected static responses.

**Persistence:** `AstraConversation` + `AstraMessage` models. Up to 20 conversations per user. Auto-summarization when conversation exceeds threshold.

## 26. AUTH SYSTEM — DETAILED

### Login Security

Two lockout mechanisms:

1. **Email-level rate limit** (`LoginAttempt` table): 5 attempts per 15 min per email. Guards against credential stuffing.
2. **Account lockout**: After 10 failed bcrypt verifications, `lockedUntil` set 30 min ahead. Supports emailed unlock tokens.

### Cookie Security (Production)

| Cookie   | Name                            | httpOnly | Secure | SameSite |
| -------- | ------------------------------- | -------- | ------ | -------- |
| Session  | `__Secure-authjs.session-token` | true     | true   | lax      |
| Callback | `__Secure-authjs.callback-url`  | true     | true   | lax      |
| CSRF     | `__Host-authjs.csrf-token`      | true     | true   | lax      |

### API Key Authentication (v1)

- Keys stored as SHA-256 hashes in `ApiKey` model
- Bearer token auth (`Authorization: Bearer ck_...`)
- Scope-based authorization (JSON array)
- Optional HMAC request signing (`X-Signature` header)
- Per-key configurable rate limits
- Full request logging to `ApiRequestLog`

### Graceful Degradation

If `AUTH_SECRET` is not set, auth module exports no-op stubs — public pages and assessment wizard work without auth.

### Architecture Notes

1. **NextAuth beta in production**: `next-auth@5.0.0-beta.30` — pre-release version carries stability risk
2. **`unsafe-inline` in CSP**: Required due to Next.js 15 limitation (nonce not applied to generated `<script>` tags). Weakens XSS protection.
3. **Cron schedule overlaps**: `onboarding-emails` and `churn-detection` both at 10:00 UTC. `compute-rrs` and `regulatory-feed` both at 7:00 UTC. Not necessarily problematic but worth monitoring.
4. **AI model hardcoded**: `GENERATION_MODEL` defaults to `claude-sonnet-4-6` — needs updating as Claude versions evolve.
5. **Per-org encryption**: Each tenant's sensitive data uses a unique key derived from master key + organizationId via scrypt. LRU cache with 100 entries.
6. **Audit trail hash chain**: `AuditLog` stores `entryHash` (SHA-256 of current) and `previousHash` (previous entry), forming tamper-evident chain. `/api/audit/verify` validates integrity.
7. **Middleware size**: Kept under 1MB for Vercel Edge Function limit. All security logic (CSP, CSRF, rate limiting, bot detection, auth guards) runs at the edge.
